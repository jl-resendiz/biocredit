# Results JSON Schema

This directory contains machine-readable model results in JSON format.
Figure scripts read from these files instead of hardcoding model values.

**Pipeline flow:**

```
scripts/run_abm.py                              -> abm_results.json
src/model/glv_formation.py                       -> glv_results.json
scripts/robustness_sensitivity.py                -> sensitivity_results.json
scripts/robustness_demand_functions.py           -> demand_robustness_results.json

src/figures/fig2_mechanism.py    <- reads abm_results.json + empirical CSV
src/figures/fig3_policy.py       <- reads sensitivity_results.json + demand_robustness_results.json
src/figures/suppfig1_robustness.py <- reads glv_results.json + abm_results.json
```

fig1_problem.py reads directly from CSV (empirical only, no model values).

## Regeneration

Run `make results` to regenerate all JSON files from model runs:

```bash
make results    # runs abm, glv
make figures    # reads from JSON (or fallback if JSON not found)
make all        # results + figures + verify
```

## Notes

- JSON files are in `.gitignore` (regenerated by running models).
- Figure scripts have fallback values if JSON files are missing.
- Each JSON includes a timestamp and parameter snapshot for auditability.
